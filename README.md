# ML--Assignment2-IEEE-CIS-Fraud-Detection
# პროექტის სათაური: IEEE-CIS Fraud Detection 

მიზანია ელექტრონული ტრანზაქციების მონაცემებზე დაყრდნობით თაღლითური (Fraud) ტრანზაქციების გამოვლენა. მონაცემთა ბაზა შეიცავს როგორც ტრანზაქციის დეტალებს, ასევე მომხმარებლის იდენტიფიკაციის მონაცემებს. მთავარი გამოწვევა არის კლასების დისბალანსი, დიდი მოცულობის მონაცემები და ანონიმიზირებული ცვლადები.

### Cleaning მიდგომები და მონაცემების მომზადება
* **მეხსიერების ოპტიმიზაცია (Memory Reduction):** ვინაიდან IEEE მონაცემთა ბაზა საკმაოდ დიდია, დავწერეთ `reduce_mem_usage` ფუნქცია, რომელიც ამოწმებს თითოეული სვეტის მნიშვნელობების დიაპაზონს და შესაბამისად ამცირებს მონაცემთა ტიპებს (მაგ. `float64` გადაყავს `float32` ან `float16`-ში). ამან მნიშვნელოვნად შეამცირა RAM-ის მოხმარება და აგვარიდა Memory Error-ები.
* **Train/Test Split:** სანამ რაიმე სახის ტრანსფორმაციას (მაგ. WOE) ჩავატარებდით, მონაცემები გავყავით Train და Test ნაწილებად (80/20 პროპორციით, `stratify=y` პარამეტრით). ეს კრიტიკულად მნიშვნელოვანია, რათა თავიდან ავიცილოთ მონაცემების გაჟონვა (Data Leakage) და მოდელმა არ "დაისწავლოს" ინფორმაცია სატესტო სეტიდან.

### Feature Engineering
* **Nan მნიშვნელობების დამუშავება:** 
  * კატეგორიული ცვლადებისთვის გამოვიყენეთ ყველაზე ხშირად განმეორებადი მნიშვნელობა (Mode).
  * რიცხვითი ცვლადებისთვის გამოვიყენეთ მედიანა (Median), რადგან ის ნაკლებად მგრძნობიარეა ექსტრემალური მნიშვნელობების (Outliers) მიმართ.
* **კატეგორიული ცვლადების რიცხვითში გადაყვანა:** 
  * შევქმენით `CustomPreprocessor` კლასი. 
  * დავთვალეთ თითოეული კატეგორიული ცვლადის უნიკალური მნიშვნელობების რაოდენობა.
  * **One-Hot Encoding:** ის ცვლადები, რომელთა უნიკალური მნიშვნელობების რაოდენობა იყო 3 ან ნაკლები, გადავიყვანეთ One-Hot ენკოდინგით.
  * **Weight of Evidence (WOE) Encoding:** მაღალი კარდინალობის მქონე კატეგორიული ცვლადებისთვის (3-ზე მეტი უნიკალური მნიშვნელობა) გამოვიყენეთ WOE ენკოდინგი. ამ მეთოდმა საშუალება მოგვცა კატეგორიები ჩაგვენაცვლებინა მათი სტატისტიკური მნიშვნელობით თაღლითობის გამოვლენის კუთხით.
  * **Information Value (IV):** WOE ენკოდინგის პროცესში დავითვალეთ Information Value (IV) თითოეული მახასიათებლისთვის. აღმოჩნდა, რომ `ProductCD` არის ძალიან ძლიერი პრედიქტორი (IV > 0.5), ხოლო ცვლადები, როგორიცაა `card4`, თითქმის გამოუსადეგარია (IV < 0.02).
